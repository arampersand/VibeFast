## Aplicar migrations

Es el mismo flujo del [Paso 6 del Quick start](/docs/setup/quick-start#paso-6--sube-el-schema-a-supabase). Si tu proyecto ya está conectado (`supabase link` ya corrido), basta con:

```bash
supabase db push
```

Si es la primera vez en esta compu, antes corre `supabase login` y `supabase link --project-ref <tu-project-ref>` — el `<tu-project-ref>` está en la URL del <Term id="dashboard">dashboard</Term> (`https://supabase.com/dashboard/project/TU-REF`) o en **Project Settings → General**.

## Configurar redirect URLs

En <Term id="supabase">Supabase</Term> → Authentication → URL Configuration:

- **Site URL**: tu URL de <Term id="vercel">Vercel</Term> (`https://<tu-producto>.vercel.app` o tu <Term id="dominio">dominio</Term>).
- **Redirect URLs**: agrega `https://<tu-producto>.vercel.app/auth/callback`.

En los dos, cambia `https://<tu-producto>.vercel.app` por la URL real que te dio Vercel al <Term id="deploy">deployar</Term> (o tu dominio propio si ya lo conectaste).

## Concepto: MCP

MCP (Model Context Protocol) es un **estándar abierto** para conectar IA con herramientas y datos. En VibeFast lo usamos doble:

- Tu app **expone** sus tools vía MCP (`/api/mcp`) — Claude Desktop, otros clientes pueden usar tu producto como tool.
- Tu app **consume** MCPs externos — Gmail, Notion, ESP-Claw del [módulo de hardware (Semanas 8–9)](/docs/tutoriales/semana-8-hardware-fundamentos).

## Prepárate desde ahora (esto sí puedes hacerlo hoy)

1. **Termina la Semana 4**: necesitas al menos UNA tool propia funcionando en `/chat` — el agente y el MCP se construyen encima de ese registry.
2. **Prueba el agente de ejemplo**: el boilerplate ya trae un flujo `recuperar → decidir → actuar` (`web/lib/agents/examples/recoverDecideAct.js` + endpoint `/api/ai/agent`). Pídele a Cursor: *"muéstrame cómo probar el endpoint /api/ai/agent con un ejemplo y explícame el grafo paso a paso como si fuera nuevo en esto"*.
3. **Aterriza tu caso**: escribe en tu ficha qué tarea de varios pasos haría un agente en TU producto (ej. "recibe un lead → lo clasifica → responde → agenda"). Esa es la que construirás en vivo.
4. Si quieres adelantar el concepto MCP con las manos: instala **Claude Desktop** y conéctale un MCP público cualquiera, para sentir qué es un cliente MCP.
